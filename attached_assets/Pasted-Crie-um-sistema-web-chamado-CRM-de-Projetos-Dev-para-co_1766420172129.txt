Crie um sistema web chamado “CRM de Projetos (Dev)” para controlar projetos de desenvolvimento com foco em Kanban, prazos e relatórios.

1) Autenticação, cadastro de usuários e permissões

Habilite autenticação e permita cadastro/gestão de usuários (CRUD).

Perfis (roles):

Admin: acesso total.

Gestor: cria/edita projetos, configura formulários padrão, vê relatórios.

Técnico: vê e atualiza apenas projetos/cards atribuídos a ele, preenche formulário do card, vê prazos dos seus projetos.

Regras de acesso:

Somente Admin e Gestor podem criar/editar projetos e modelos de formulário.

Técnico não pode ver senhas/credenciais de clientes se não estiver autorizado (criar controle por “visível para: Admin/Gestor/Técnicos autorizados”).

3) Regra principal (obrigatória)

Todo projeto deve ter um Formulário Padrão.

Na criação do projeto, exigir selecionar um FormTemplate existente ou criar um novo na hora (wizard).

Não permitir criar projeto sem form_template_padrao_id.

Ao criar um Card dentro de um projeto:

criar automaticamente um CardFormResponses já vinculado ao form_template_padrao_id do projeto

pré-criar registros de CardFormAnswers para cada campo do template (ou gerar sob demanda ao abrir o formulário)

o formulário do card deve ser exibido como uma aba/seção “Formulário Padrão” dentro do card.

7) Relatórios

Criar a página “Relatórios” com:

Quantidade de projetos por técnico (tabela + gráfico)

Total de projetos

Projetos dentro do prazo:

regra: hoje <= data_prevista_entrega e status != Concluído (ou concluído antes do prazo)

Projetos fora do prazo:

regra: hoje > data_prevista_entrega e status != Concluído

Filtros por período (data_inicio), cliente, técnico, status.

8) Alertas automáticos (projeto fora do prazo)

Criar automações/workflows:

Ao abrir o sistema e/ou diariamente:

identificar projetos com hoje > data_prevista_entrega e status != Concluído

criar/atualizar registro em Alerts (se ainda não existir aberto)

setar alerta_atraso_ativo = true

notificar Gestor/Admin (e técnico responsável) dentro do app (e-mail/integração se existir)

Quando o projeto for marcado como Concluído:

resolver alerta aberto e setar alerta_atraso_ativo = false
